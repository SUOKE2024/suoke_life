# 索克生活系统架构设计

## 1. 系统概述

### 1.1 设计目标
- 高可用性：系统 99.99% 可用性
- 高性能：API 响应时间 < 100ms
- 高扩展性：支持水平扩展
- 安全可靠：多层次安全防护
- 容灾备份：多云架构保障

### 1.2 技术选型
- 客户端：Flutter 3.16+
- 服务端：FastAPI + Flask + Nginx
- 数据库：MySQL 8.0 + Redis + SQLite
- AI引擎：豆包大模型（三模型配置）
  - doubao-pro-32k (ep-20241203122456-lfzdw)：通用对话与智能助手交互
  - doubao-pro-128k (ep-20241203122647-5dvrv)：复杂场景与长文本处理
  - doubao-embedding (ep-20241203122853-dmwdj)：文本向量化与语义理解
- 云服务：阿里云 + 腾讯云双云架构

|   客户端层      |     |    应用层      |     |    数据层      |

## 9. 云服务架构

### 9.1 阿里云服务
- ECS：主要计算资源
  - 规格：ecs.e-c1m1.large
  - 用途：核心业务服务器
- RDS MySQL：主数据库服务
- OSS：对象存储服务
- SLB：负载均衡服务
- 专有网络VPC配置

### 9.2 腾讯云服务
- CVM：备份计算资源
  - 规格：标准型 S5
  - 用途：容灾备份服务器
- TencentDB：备份数据库服务
- COS：对象存储服务
- CLB：负载均衡服务
- 私有网络VPC配置

### 9.3 多云协同

- 智能负载均衡
- 容灾切换机制
- 统一监控管理
