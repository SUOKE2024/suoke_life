# 安全指南

## JWT 认证

### 概述
系统使用JWT（JSON Web Token）进行用户认证和授权。JWT包含用户身份信息，并通过数字签名保证其完整性。

### 实现细节
- 使用HS256算法进行签名
- 密钥存储在环境变量中
- Token有效期：7天
- 包含以下声明：
  - iss: 签发者
  - sub: 用户ID
  - jti: 唯一标识符
  - exp: 过期时间

### 安全建议
1. 在生产环境中使用强密钥
2. 定期轮换密钥
3. 使用HTTPS传输Token
4. 设置合理的Token有效期
5. 实现Token刷新机制

## 其他安全措施

- 请求速率限制
- 输入验证
- 安全头设置
- 定期安全审计
