# 索克生活APP前端架构设计

## 概述

索克生活APP前端采用Flutter跨平台框架构建，遵循Clean Architecture架构原则，实现高性能、可维护、响应式的用户界面。前端架构设计旨在支持复杂的中医健康管理业务需求、多智能体交互以及丰富的多模态数据展示，同时保证跨平台一致性和出色的用户体验。

### 1. 表现层 (Presentation Layer)

负责UI渲染和用户交互，采用MVVM (Model-View-ViewModel) 模式：

- **屏幕 (Screens)**：完整页面，组合多个组件
- **视图模型 (ViewModels)**：使用Riverpod StateNotifier管理UI状态和业务逻辑
- **组件 (Widgets)**：可复用UI组件，专注于渲染
- **路由 (Routes)**：导航逻辑和路由配置

#### 设计模式与最佳实践

- **状态提升**：将共享状态提升到合适的视图模型
- **单向数据流**：从视图模型到UI的单向数据流
- **响应式UI**：UI根据状态变化自动更新
- **组件分解**：将大型UI分解为可管理的小组件
- **状态隔离**：不同模块状态相互隔离

### 2. 领域层 (Domain Layer)

包含核心业务逻辑和规则，与具体实现无关：

- **实体 (Entities)**：核心业务模型，表示业务概念
- **用例 (Use Cases)**：代表应用程序特定的业务规则
- **仓库接口 (Repository Interfaces)**：定义数据操作契约
- **服务接口 (Service Interfaces)**：定义业务服务契约

#### 关键设计原则

- **业务逻辑集中**：领域逻辑集中在用例中
- **平台无关**：不依赖任何Flutter框架代码
- **接口抽象**：通过接口定义数据操作
- **实体完整性**：实体具有完整业务规则

### 3. 数据层 (Data Layer)

负责数据获取和持久化：

- **仓库实现 (Repository Implementations)**：实现领域层定义的仓库接口
- **数据源 (Data Sources)**：本地和远程数据源的具体实现
- **数据模型 (Data Models)**：外部数据结构的表示
- **映射器 (Mappers)**：在数据模型和领域实体间转换

#### 数据流管理

- **缓存策略**：实现适当的数据缓存机制
- **离线优先**：支持离线操作和同步
- **错误处理**：统一错误处理和恢复机制
- **数据转换**：确保数据模型到实体的安全转换

### 4. 核心层 (Core Layer)

提供通用功能和工具：

- **网络 (Network)**：HTTP客户端、拦截器
- **存储 (Storage)**：数据库辅助工具、偏好设置管理
- **工具 (Utils)**：辅助函数、扩展方法
- **配置 (Config)**：环境配置、常量

#### 辅助工具与实用程序

- **日志系统**：结构化日志记录
- **异常处理**：全局异常捕获与处理
- **格式化工具**：日期、货币等格式化
- **验证工具**：输入验证辅助方法

## UI架构与组件设计

### 组件化UI设计

索克生活APP采用原子设计方法论构建UI组件库：

1. **原子 (Atoms)**：基础UI元素，如按钮、输入框、标签
2. **分子 (Molecules)**：由多个原子组合的功能单元，如搜索框、卡片
3. **有机体 (Organisms)**：复杂UI组件，如头部导航、列表项
4. **模板 (Templates)**：页面布局框架
5. **页面 (Pages)**：完整用户界面

### 组件分类

- **基础组件**：通用UI元素，跨应用复用
- **功能组件**：特定功能实现，如四诊采集组件
- **业务组件**：特定业务逻辑组件，如体质类型卡
- **布局组件**：页面结构和布局

### 主题系统

- **分层主题定义**：核心主题→模块主题→组件主题
- **动态主题切换**：支持明暗模式和季节主题
- **响应式主题**：根据设备设置自动调整
- **中医特色主题**：融入中医五行色彩系统

### 自适应与响应式设计

- **布局构建器**：使用`LayoutBuilder`实现响应式布局
- **自适应断点**：定义手机、平板、桌面断点
- **弹性布局**：结合Flex、Expanded等实现弹性UI
- **响应式组件**：根据屏幕尺寸调整行为

## 状态管理详解

### Riverpod 架构

索克生活APP采用Riverpod作为主要状态管理解决方案：

- **Provider**：声明式依赖提供
- **StateNotifier**：不可变状态容器
- **ChangeNotifier**：适用于复杂局部状态
- **StreamProvider**：响应式数据流

### 状态分类

1. **应用状态**：全局共享状态，如用户认证状态
2. **导航状态**：当前路由和导航历史
3. **会话状态**：临时会话数据，如当前交互上下文
4. **模块状态**：特定功能模块的状态
5. **页面状态**：单个页面的局部状态
6. **UI状态**：加载、错误、表单状态等

### 状态管理最佳实践

- **状态隔离**：不同模块状态相互隔离
- **单一数据源**：状态有明确的单一来源
- **不可变状态**：使用不可变对象表示状态
- **选择性重建**：使用`select`优化重建范围
- **状态持久化**：关键状态持久化到存储

### 状态同步

- **后台同步服务**：处理状态持久化和同步
- **乐观更新**：先更新UI，后台执行实际操作
- **冲突解决策略**：定义冲突解决规则
- **状态回滚机制**：失败时恢复先前状态
