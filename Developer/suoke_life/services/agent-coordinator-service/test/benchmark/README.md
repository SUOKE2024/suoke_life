# 索克生活API性能基准测试工具

这个工具用于测量索克生活API的性能基准，并支持与历史基准数据比较。它能够帮助团队识别性能退化，确保API保持高效运行。

## 功能特点

- 支持多端点并发测试
- 详细的性能指标收集（响应时间、成功率等）
- 与历史基准数据比较
- 性能阈值检查
- 美观的控制台输出
- 详细的JSON报告生成

## 安装

在`services/agent-coordinator-service/test/benchmark`目录下运行：

```bash
npm install
```

## 使用方法

### 基本使用

```bash
# 使用默认配置运行基准测试
npm run benchmark

# 设置当前结果为新基准
npm run benchmark:baseline

# 指定配置文件运行
npm run benchmark -- --config custom-config.json

# 指定API基础URL
npm run benchmark -- --url http://api.suoke.life/api
```

### 命令行选项

- `-c, --config <path>`: 配置文件路径
- `-u, --url <url>`: API基础URL
- `-i, --iterations <number>`: 每个端点的迭代次数
- `-n, --concurrency <number>`: 并发请求数
- `-o, --output <dir>`: 结果输出目录
- `-b, --baseline <path>`: 基准文件路径
- `-s, --set-baseline`: 将当前结果设置为新基准
- `-t, --token <token>`: API认证令牌

## 配置文件

配置文件使用JSON格式，包含以下字段：

```json
{
  "apiBaseUrl": "http://localhost:4000/api",
  "iterations": 50,
  "concurrency": 10,
  "endpoints": [
    {
      "name": "知识搜索",
      "method": "GET",
      "path": "/knowledge/search?query=中医",
      "payload": null
    },
    {
      "name": "智能体查询",
      "method": "POST",
      "path": "/agents/default/query",
      "payload": {
        "query": "中医养生有哪些方法？",
        "userId": "user-123"
      }
    }
  ],
  "outputDir": "./benchmark-results",
  "threshold": {
    "p50": 500,
    "p90": 800,
    "p95": 1000
  }
}
```

字段说明：
- `apiBaseUrl`: API的基础URL
- `iterations`: 每个端点的测试迭代次数
- `concurrency`: 并发请求数
- `endpoints`: 要测试的端点数组
  - `name`: 端点名称（用于显示和基准比较）
  - `method`: HTTP方法（GET, POST等）
  - `path`: API路径（相对于baseUrl）
  - `payload`: 请求负载（对于POST、PUT等）
- `outputDir`: 结果输出目录
- `threshold`: 性能阈值，用于判断测试是否通过
  - `p50`: 中位数响应时间阈值（毫秒）
  - `p90`: 90%分位数响应时间阈值（毫秒）
  - `p95`: 95%分位数响应时间阈值（毫秒）

## CI/CD集成

这个基准测试工具可以集成到CI/CD管道中。当性能阈值检查失败时，工具会返回非零退出代码，触发CI/CD失败。

示例`.gitlab-ci.yml`配置：

```yaml
stages:
  - test
  - benchmark

performance_benchmark:
  stage: benchmark
  script:
    - cd services/agent-coordinator-service/test/benchmark
    - npm install
    - npm run benchmark
  only:
    - main
    - tags
```

## 定期性能监控

建议设置定期执行基准测试的计划任务，以监控系统随时间的性能变化。例如，可以每天或每周执行一次基准测试，并将结果保存至指定位置。

## 故障排除

1. 如果遇到网络问题，请检查API是否可达，并确认基础URL配置正确。
2. 如果基准比较显示性能大幅下降，请检查最近的代码变更和服务器负载情况。
3. 确保测试环境尽可能接近生产环境，以获得准确的性能指标。
