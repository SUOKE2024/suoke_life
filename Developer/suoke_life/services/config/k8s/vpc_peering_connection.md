# 阿里云VPC对等连接配置指南

## 概述

本文档提供了在阿里云上配置VPC对等连接的详细步骤，用于解决主服务器（118.31.223.213）与Kubernetes集群（120.26.161.52）之间的网络连接问题。

## 目标环境

- **主服务器VPC**: 
  - IP地址: 118.31.223.213
  - VPC ID: vpc-bp1xxxxxxxx（示例，需替换为实际值）
  - CIDR块: 172.16.169.0/24（示例，需替换为实际值）
  - 地域: 华东1（杭州）

- **Kubernetes集群VPC**:
  - IP地址: 120.26.161.52
  - VPC ID: vpc-bp2xxxxxxxx（示例，需替换为实际值）
  - CIDR块: 172.16.0.0/12（示例，需替换为实际值）
  - 地域: 华东1（杭州）

## 解决方案

### 方案一：使用云企业网（CEN）连接VPC

云企业网（Cloud Enterprise Network）是阿里云提供的全球网络连接服务，用于在不同VPC之间建立私有连接。

#### 步骤1：创建云企业网实例

1. 登录[阿里云控制台](https://console.aliyun.com/)
2. 导航到**云企业网**服务
3. 点击**创建云企业网实例**
4. 填写以下信息:
   - 名称: `suoke-vpc-connection`
   - 描述: `索克生活项目VPC间通信配置`
5. 点击**确定**创建实例

#### 步骤2：加入网络实例

1. 在云企业网控制台，选择刚创建的实例
2. 点击**网络实例管理** > **加入网络实例**
3. 加入主服务器VPC:
   - 账号: 当前账号
   - 资源类型: 专有网络(VPC)
   - 地域: 华东1（杭州）
   - 资源: 选择主服务器所在的VPC
4. 重复上述步骤，加入Kubernetes集群VPC

#### 步骤3：配置带宽包（如需跨地域）

如果两个VPC位于同一地域，可以跳过此步骤。如果位于不同地域，需要购买带宽包:

1. 在云企业网控制台，选择**带宽包管理**
2. 点击**购买带宽包**
3. 选择互通区域，并设置所需带宽

#### 步骤4：检查路由表

1. 在云企业网控制台，选择**路由管理**
2. 验证是否有到对方VPC的路由条目，默认情况下系统会自动学习路由

### 方案二：使用VPC对等连接

如果不想使用云企业网，可以使用高速通道的VPC对等连接功能。

#### 步骤1：创建对等连接

1. 登录[阿里云控制台](https://console.aliyun.com/)
2. 导航到**高速通道** > **VPC对等连接** > **发起端**
3. 点击**创建对等连接**
4. 填写以下信息:
   - 连接场景: VPC互连
   - 创建方式: 跨账号（如果两个VPC在同一账号下，选择同账号）
   - 地域: 选择VPC所在地域
   - 接收端地域: 选择对端VPC所在地域
   - 接收端VPC: 填写对端VPC的ID
   - 带宽值: 根据需要设置，如100Mbps

#### 步骤2：接受对等连接请求

如果是跨账号连接，需要在接收端账号中接受连接请求:

1. 在接收端账号的阿里云控制台中，导航到**高速通道** > **VPC对等连接** > **接收端**
2. 找到待处理的连接请求并接受

#### 步骤3：配置路由

两端VPC均需要配置指向对方VPC的路由:

1. 在阿里云控制台中，导航到**专有网络VPC** > **路由表**
2. 选择需要配置的VPC的路由表
3. 点击**添加路由条目**
4. 填写以下信息:
   - 目标网段: 对端VPC的CIDR块（例如172.16.0.0/12）
   - 下一跳类型: 选择"VPC对等连接"
   - 下一跳: 选择已创建的对等连接实例

### 方案三：使用VPN网关

如果前两种方法不适用，可以考虑使用IPsec VPN连接两个VPC。

#### 步骤1：创建VPN网关

在两个VPC中分别创建VPN网关:

1. 在阿里云控制台中，导航到**VPN网关**
2. 点击**创建VPN网关**
3. 选择VPC和规格，完成创建

#### 步骤2：配置用户网关

1. 在VPN网关控制台，点击**用户网关** > **创建用户网关**
2. 输入对端VPN网关的公网IP地址

#### 步骤3：创建IPsec连接

1. 在VPN网关控制台，点击**IPsec连接** > **创建IPsec连接**
2. 配置预共享密钥和IPsec策略
3. 配置本端和对端网络
4. 在对端VPN网关上执行类似配置

## 安全组配置

完成网络连接后，还需要配置安全组规则，允许两个VPC之间的流量:

### 主服务器安全组规则

1. 添加入站规则，允许来自Kubernetes集群的流量:
   - 协议类型: 全部
   - 端口范围: -1/-1（所有端口）
   - 源IP地址: 120.26.161.52/32 和 Kubernetes集群VPC的CIDR块

### Kubernetes集群安全组规则

1. 添加入站规则，允许来自主服务器的流量:
   - 协议类型: 全部
   - 端口范围: -1/-1（所有端口）
   - 源IP地址: 118.31.223.213/32 和 主服务器VPC的CIDR块

## 验证连接

完成配置后，验证连接是否成功:

1. 在主服务器上执行:
   ```bash
   ping 120.26.161.52
   ```

2. 测试特定服务的连接:
   ```bash
   telnet 120.26.161.52 80
   curl -v http://120.26.161.52/api/health
   ```

## 故障排查

如果连接仍然失败，请检查以下几点:

1. **路由表配置**: 确认两个VPC的路由表中有指向对方VPC的路由条目
2. **安全组规则**: 确认安全组允许必要的流量
3. **网络ACL**: 如果使用了网络ACL，确保其允许必要的流量
4. **系统防火墙**: 检查操作系统级别的防火墙规则
5. **日志分析**: 检查VPC流日志，排查流量是否被拦截
