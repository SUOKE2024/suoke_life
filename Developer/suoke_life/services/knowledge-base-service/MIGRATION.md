# 知识库服务 (Knowledge Base Service) 迁移文档

本文档记录了知识库服务从Node.js到Go的迁移过程和相关信息。

## 迁移概述

知识库服务已经从原来的Node.js实现迁移到Go语言实现，以提高性能、可靠性和可维护性。迁移涉及以下主要方面：

1. 后端服务从Node.js重写为Go
2. 数据库从MongoDB更改为PostgreSQL
3. 添加Milvus向量数据库支持
4. 更新Docker和Kubernetes配置
5. 更新CI/CD流程

## 架构变更

### 技术栈变更
- **语言**: 从Node.js/TypeScript迁移到Go
- **主数据库**: 从MongoDB迁移到PostgreSQL
- **向量存储**: 新增Milvus向量数据库
- **缓存**: 去除Redis依赖，利用Go内置的缓存机制

### 代码结构
Go服务采用了清晰的领域驱动设计(DDD)结构：
```
knowledge-base-service/
├── cmd/              # 应用入口点
├── config/           # 配置管理
├── internal/         # 内部代码
│   ├── api/          # API定义
│   ├── domain/       # 领域模型和服务
│   ├── infrastructure/ # 基础设施代码
│   └── interfaces/   # 接口实现
├── pkg/              # 可共享的包
└── ...
```

## 功能变更

### 新增功能
1. 支持向量搜索能力，提高搜索准确性
2. 支持中文文本分块（智能分段）
3. 支持文档版本控制
4. 新增指标监控端点

### 改进的功能
1. 更高效的文本处理
2. 更好的并发处理能力
3. 更低的内存占用
4. 更快的启动和响应时间

## 部署变更

### Docker配置
更新了Dockerfile，适配Go服务部署需求，采用多阶段构建减小镜像大小。

### Kubernetes配置
更新了K8s的部署配置，包括：
- 更新了环境变量和配置
- 更新了健康检查路径
- 调整了资源配置

### 环境变量变更
详见`.env.example`文件，主要变更包括数据库连接配置、向量存储配置等。

## 迁移步骤

1. 理解原有Node.js服务的功能和业务逻辑
2. 设计Go服务的架构和数据模型
3. 实现Go服务的核心功能
4. 设置PostgreSQL数据库模式和Milvus集合
5. 实现数据迁移脚本
6. 更新部署配置
7. 测试新服务功能和性能
8. 切换流量到新服务

## 数据迁移

开发了专门的数据迁移工具，实现从MongoDB到PostgreSQL的数据迁移：
- 保留所有文档ID以确保向后兼容
- 重新生成所有文档的向量嵌入

## 清理指南

如果需要彻底清理旧的Node.js代码，请执行以下步骤：

1. 确保Go服务已部署并正常运行
2. 确保所有数据已迁移成功
3. 运行清理脚本：`./scripts/cleanup-nodejs.sh`

## 注意事项

1. API路径前缀从`/`更改为`/api/v1/`
2. 健康检查端点从`/health`更改为`/api/v1/health`
3. 所有配置都应该通过环境变量或配置文件设置
