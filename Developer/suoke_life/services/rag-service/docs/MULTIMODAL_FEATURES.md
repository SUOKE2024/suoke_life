# 索克生活 RAG 服务多模态支持

## 功能概述

索克生活 RAG 服务的多模态支持模块为应用提供了以下关键功能：

1. **中医图像分析与检索**：
   - 舌诊图像分析与特征提取
   - 面诊图像分析与特征提取
   - 基于图像的相似案例检索
   - 舌诊图像特征增强向量表示

2. **中医音频分析与检索**：
   - 方言音频分析
   - 脉诊音频分析
   - 基于音频的相似案例检索
   - 音频特征增强向量表示

3. **混合模态搜索**：
   - 文本+图像联合检索
   - 文本+音频联合检索
   - 多模态权重动态调整

4. **TCM特色功能**：
   - 舌诊特征映射到中医证型
   - 面诊特征映射到脏腑关系
   - 方言特征辅助诊断

## 技术实现

### 1. 多模态嵌入器

系统设计了统一的嵌入器接口层次结构：

```
Embedder (基础文本嵌入)
  ├── ImageEmbedder (图像嵌入)
  │     └── TCMImageEmbedder (中医图像嵌入)
  ├── AudioEmbedder (音频嵌入)
  │     └── TCMAudioEmbedder (中医音频嵌入)
  └── MultiModalEmbedder (多模态嵌入)
```

每种嵌入器都提供了特定的功能：

- **TCMImageEmbedder**: 专门针对中医图像进行处理，包括舌诊和面诊图像分析
- **TCMAudioEmbedder**: 专门处理中医相关音频，如脉象音频和方言音频
- **MultiModalEmbedder**: 提供跨模态嵌入和检索功能

### 2. 中医图像特征提取

舌诊图像处理流程：

1. 图像分类 (舌诊/面诊/其他)
2. 舌体分割
3. 舌质颜色分析
4. 舌苔颜色和厚度分析
5. 舌形特征分析
6. 特征映射到中医理论

面诊图像处理流程：

1. 面部区域分割
2. 面色分析
3. 五官特征提取
4. 脏腑对应关系分析
5. 特征映射到中医理论

### 3. API接口设计

我们提供了以下REST API接口：

- `/api/search`: 标准文本搜索接口
- `/api/search/image`: 图像检索接口
- `/api/search/audio`: 音频检索接口

每个接口都支持多种参数，允许进行细粒度控制和自定义：

- 查询领域指定
- TCM特有选项
- 混合搜索与重排序
- 结果数量控制
- 缓存控制

### 4. 组件工厂集成

所有多模态组件通过组件工厂进行统一管理和访问：

```go
factoryInstance.GetImageEmbedder("tcm")
factoryInstance.GetAudioEmbedder("tcm")
factoryInstance.GetMultiModalEmbedder("default")
```

这种设计提供了良好的可扩展性，可以轻松添加新的embedder实现或替换现有实现。
