# 索克生活RAG服务最小化部署指南

本文档提供索克生活RAG服务的最小化部署方案。此方案专为验证基础功能和服务框架设计，不包含完整功能。

## 最小化部署方案设计思路

基于其他微服务成功部署的经验，我们为RAG服务设计了一个最小化部署方案，主要特点如下：

1. **精简容器设计**：使用轻量级Alpine基础镜像，只包含必要组件
2. **可靠的健康检查机制**：提供稳定可靠的健康检查端点
3. **合理的资源配置**：降低内存和CPU需求
4. **完善的部署脚本**：包含目录检查、容器构建、状态验证等关键步骤
5. **无依赖或简单依赖**：使用内存数据库模式进行测试部署

## 快速开始

### 1. 准备环境

确保已安装以下软件：
- Docker
- Docker Compose
- Go 1.18+（可选，仅本地开发需要）

### 2. 运行部署脚本

```bash
cd /path/to/suoke_life/services/rag-service
./scripts/deploy-minimal-rag.sh
```

脚本将自动执行以下操作：
- 检查并创建必要的目录
- 创建Dockerfile.minimal.rag
- 创建docker-compose.minimal.rag.yml
- 创建Nginx配置
- 部署并启动服务
- 验证服务健康状态

### 3. 验证服务

服务启动成功后，可以通过以下方式验证：

```bash
# 测试健康状态
curl http://localhost:8080/health

# 测试查询API
curl "http://localhost:8080/api/search?q=感冒的中医治疗方法"

# 或运行测试脚本
./scripts/test-minimal-rag.sh
```

## 故障排除

### 1. 服务无法启动

如果服务无法启动，请检查：
- Docker和Docker Compose是否正确安装
- 必要端口（8080, 8081）是否被占用
- 日志文件中是否有具体错误信息

### 2. 服务启动但健康检查失败

如果服务启动但健康检查失败，可能的原因：
- Go服务内部错误
- 网络配置问题
- 权限问题

尝试以下解决方案：
```bash
# 查看服务日志
docker logs suoke-rag-service-minimal

# 重启服务
docker-compose -f docker-compose.minimal.rag.yml restart
```
