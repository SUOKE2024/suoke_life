# Docker优化指南

## 镜像加速配置

为了提高Docker镜像拉取速度，我们提供了自动配置脚本`scripts/docker-mirror.sh`，该脚本会配置以下镜像加速器：

- Docker中国官方镜像
- 网易云镜像
- 百度云镜像

### 使用方法

执行以下命令：

```bash
# 添加执行权限
chmod +x scripts/docker-mirror.sh

# 运行脚本
./scripts/docker-mirror.sh
```

脚本会根据系统类型自动配置适当的镜像加速设置。

## Dockerfile优化

我们对Dockerfile进行了以下优化：

1. **多阶段构建**：使用两个阶段，第一阶段编译Go二进制文件，第二阶段创建最小运行镜像
2. **缓存优化**：
   - 将`go.mod`和`go.sum`分离复制，优先下载依赖
   - 仅在依赖变更时才重新下载依赖
3. **镜像体积优化**：
   - 使用alpine作为基础镜像
   - 编译时使用`-ldflags "-s -w"`去除调试信息减小二进制体积
   - 清理构建过程中的临时文件和缓存
4. **安全优化**：
   - 创建并使用非root用户
   - 最小化安装的包
   - 使用COPY而非ADD
5. **构建速度优化**：
   - 使用Go模块代理加速依赖下载
   - 仅复制必要的文件

## 生产环境部署

使用我们提供的`docker-compose.prod.yml`文件进行生产环境部署：

```bash
# 构建并部署
./scripts/deploy-prod.sh
```

生产环境特性：

1. 自动配置Nginx反向代理
2. 配置适当的资源限制
3. 配置健康检查
4. 配置日志轮转
5. 添加Redis缓存支持
6. 配置PostgreSQL+pgvector向量数据库

## 性能调优建议

### 容器资源限制

生产环境中已为各服务配置了合理的资源限制：

- RAG服务：CPU 2核，内存4GB
- Redis：CPU 0.5核，内存1GB
- PostgreSQL：CPU 1核，内存2GB
- Nginx：CPU 0.5核，内存512MB

根据实际负载情况，可在`docker-compose.prod.yml`中调整这些值。

### 网络优化

1. 将所有服务放在同一网络中减少网络开销
2. 使用Nginx进行负载均衡和缓存
3. 启用HTTP/2提高传输效率
4. 配置适当的超时值

### 存储优化

1. 使用数据卷持久化数据
2. 配置适当的日志轮转策略
3. 定期清理不需要的镜像和容器

### 监控和日志

1. 配置健康检查确保服务可用性
2. 使用Docker日志驱动限制日志大小
3. 配置Nginx访问日志格式包含响应时间

## 安全最佳实践

1. 使用非root用户运行容器
2. 定期更新基础镜像和依赖
3. 使用HTTPS加密通信
4. 配置适当的防火墙规则
5. 限制容器的系统调用权限

## 常见问题排查

### 镜像构建失败

检查以下可能的原因：

1. 网络连接问题 - 确认是否配置了镜像加速
2. 依赖下载错误 - 检查go.mod和go.sum是否正确
3. 构建资源不足 - 增加Docker可用资源

### 容器启动失败

1. 检查容器日志 `docker logs <container_id>`
2. 检查端口冲突 `netstat -tulpn | grep <port>`
3. 检查环境变量 `docker inspect <container_id> | grep -A 10 "Env"`

### 性能问题

1. 检查资源使用情况 `docker stats`
2. 检查日志中的慢查询
3. 使用性能分析工具如pprof分析瓶颈
