# 知识图谱服务测试指南

本文档提供了关于如何运行和维护知识图谱服务测试的详细指南。

## 测试结构

测试分为以下几个层次：

1. **单元测试**：测试各个组件的独立功能
   - 位于各个包的 `*_test.go` 文件中
   - 主要测试实体、仓库和处理程序的独立功能

2. **集成测试**：测试组件之间的交互
   - 位于 `tests/integration` 目录中
   - 测试仓库与数据库的交互
   - 测试API处理程序与仓库的交互

3. **API测试**：测试REST API接口
   - 位于 `tests/api` 目录中
   - 测试各个API端点的功能和响应

## 单元测试

### 运行单元测试

要运行所有单元测试，请执行：

```bash
go test ./internal/...
```

要运行特定包的单元测试，例如实体测试：

```bash
go test ./internal/domain/entities/...
```

要查看详细的测试输出，添加 `-v` 标志：

```bash
go test -v ./internal/...
```

## API测试

API测试用于验证服务的REST接口是否正常工作。这些测试需要一个运行中的知识图谱服务实例。

### 运行API测试

1. 首先确保知识图谱服务正在运行：

```bash
cd cmd/server
go run main.go
```

2. 在另一个终端窗口中，运行API测试：

```bash
cd tests/api
go test -v
```

### API测试文件说明

- `main_test.go`: 包含测试初始化和公共工具函数
- `node_api_test.go`: 测试基础节点API端点
- `tcm_node_api_test.go`: 测试TCM特色节点API端点
- `relationship_api_test.go`: 测试关系API端点

## 集成测试

集成测试用于验证服务的各个组件是否能正确协同工作。

### 运行集成测试

要运行所有集成测试：

```bash
cd tests/integration
go test -v
```

## 性能测试

性能测试用于评估服务在高负载下的性能表现。

### 运行性能测试

```bash
node scripts/performance-test.js
```

性能测试脚本会生成负载并输出性能指标。

## 测试覆盖率

要生成测试覆盖率报告：

```bash
go test -coverprofile=coverage.out ./...
go tool cover -html=coverage.out -o coverage.html
```

然后在浏览器中打开 `coverage.html` 文件查看覆盖率报告。

## 测试数据

测试使用的数据可以分为以下几类：

1. **模拟数据**：单元测试中使用的内存数据
2. **测试数据库**：集成测试使用的临时数据库
3. **API测试数据**：API测试过程中创建和销毁的数据

在API测试中，每个测试都会创建自己需要的数据，并在测试完成后清理这些数据，以确保测试的独立性。

## 常见问题

### 测试失败：服务未运行

如果遇到以下错误：

```
警告: API服务似乎未运行在 http://localhost:8080/api/v1，测试可能会失败
```

请确保知识图谱服务已经启动。

### 测试失败：权限问题

如果遇到权限相关错误，请确保：

1. Neo4j数据库运行中并且可以访问
2. 服务的配置正确
3. 必要的环境变量已设置

## 添加新测试

添加新测试时，请遵循以下准则：

1. 为每个新功能添加相应的单元测试
2. 为每个API端点添加API测试
3. 确保测试能够独立运行且不依赖特定环境
4. 测试应清理它创建的任何数据

## 持续集成

测试已集成到CI/CD流程中。每次提交代码后，所有测试都会自动运行。
