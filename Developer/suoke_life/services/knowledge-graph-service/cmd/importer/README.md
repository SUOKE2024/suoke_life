# 中医知识图谱数据导入工具

本工具用于将各种中医数据导入到知识图谱系统中，支持多种数据格式和中医知识类型。

## 功能特点

- 支持多种数据格式：CSV、TXT（未来会支持JSON、Excel等）
- 支持多种中医知识类型：中药、方剂、症状、疾病
- 支持批量导入和关系创建
- 提供详细的导入统计和错误报告
- 支持干运行模式，用于测试数据结构而不实际导入
- 支持跳过重复数据
- 使用Neo4j作为图数据库存储

## 使用方法

### 基本用法

```bash
go run main.go --source <数据源文件或目录> --type <数据类型>
```

### 命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--source` | 数据源文件或目录路径 | 必填 |
| `--type` | 数据类型 (herbs, formulas, symptoms, diseases) | herbs |
| `--db-uri` | Neo4j数据库URI | neo4j://localhost:7687 |
| `--db-user` | Neo4j数据库用户名 | neo4j |
| `--db-password` | Neo4j数据库密码 | password |
| `--db-database` | Neo4j数据库名称 | neo4j |
| `--batch-size` | 批处理大小 | 100 |
| `--dry-run` | 是否仅测试不实际导入 | false |
| `--skip-duplicates` | 是否跳过重复数据 | true |
| `--create-relations` | 是否创建关系 | true |
| `--log-level` | 日志级别 (debug, info, warn, error) | info |

### 示例

导入中药数据：

```bash
go run main.go --source ../../data/examples/herbs.csv --type herbs
```

导入方剂数据（干运行模式，不实际导入）：

```bash
go run main.go --source ../../data/examples/formulas.csv --type formulas --dry-run
```

## 数据格式说明

### 中药数据格式 (CSV)

中药CSV文件需要包含以下字段（字段名需与代码中的fieldMapping匹配）：

- 名称：中药名称（必填）
- 拉丁名：拉丁名
- 英文名：英文名
- 学名：学名
- 别名：别名，用分号分隔
- 分类：分类（必填）
- 来源：来源
- 性：性质，用分号分隔
- 味：味道，用分号分隔
- 归经：归经，用分号分隔
- 功效：功效，用分号分隔
- 主治：主治/应用，用分号分隔
- 用量：用量
- 描述：描述
- 标签：标签，用分号分隔
- 相关中药：相关中药，用分号分隔
- 禁忌：禁忌，用分号分隔
- 副作用：副作用，用分号分隔

### 方剂数据格式 (CSV)

方剂CSV文件需要包含以下字段：

- 名称：方剂名称（必填）
- 英文名：英文名
- 出处：出处
- 分类：分类
- 组成：组成药物，格式：药物名:用量;药物名:用量（必填）
- 功效：功效，用分号分隔
- 主治：主治适应症，用分号分隔
- 用法用量：用法用量
- 描述：描述
- 炮制：炮制方法
- 理论依据：理论依据
- 禁忌：禁忌，用分号分隔
- 加减：加减化裁
- 相关方剂：相关方剂，用分号分隔
- 标签：标签，用分号分隔

## 数据关系说明

导入工具会自动创建以下关系：

### 中药之间的关系
- SIMILAR_TO：相似关系，基于"相关中药"字段

### 方剂之间的关系
- RELATED_TO：相关关系，基于"相关方剂"字段

### 方剂与中药的关系
- CONTAINS：包含关系，基于"组成"字段，包含用量属性

## 构建与运行

### 从源代码构建

```bash
cd cmd/importer
go build -o tcm-importer
```

### 运行编译后的程序

```bash
./tcm-importer --source ../../data/examples/herbs.csv --type herbs
```

## 注意事项

1. 数据导入前建议先使用`--dry-run`参数进行测试
2. 大批量数据导入时，建议使用较大的`--batch-size`值提高性能
3. 如果数据中包含中药与方剂的交叉引用，建议先导入中药数据，再导入方剂数据
4. 导入前确保Neo4j数据库已正确配置并创建了必要的约束和索引
