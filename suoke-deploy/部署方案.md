# 索克生活APP后端部署方案

## 1. 概述

本文档详细说明索克生活APP后端微服务的部署方案，包括镜像构建、持久化存储和监控体系设置。方案设计遵循Clean Architecture + MVVM模式，确保系统的可维护性、可扩展性和可靠性。

## 2. 基础架构

### 2.1 命名空间规划

| 命名空间  | 用途           | 状态 |
|---------|---------------|------|
| suoke   | 开发环境        | 已创建 |
| suoke-dev | 开发测试环境  | 已创建 |
| monitoring | 监控系统    | 待创建 |
| logging | 日志系统       | 待创建 |

### 2.2 节点标签规划

节点按照服务类型进行了标签分类，确保资源合理分配：

- **core-service**: 核心服务节点，部署API网关、认证服务等
- **data-service**: 数据服务节点，部署数据库、缓存等
- **ai-service**: AI服务节点，部署智能体服务、RAG服务等

## 3. 构建与部署流程

### 3.1 镜像构建

项目采用基于Docker的多架构镜像构建策略，支持AMD64和ARM64两种架构。镜像构建采用BuildX工具，构建脚本位于`suoke-deploy/docker/build-images.sh`。

#### 构建流程示例

```bash
# 登录阿里云容器镜像服务
docker login --username=netsong@sina.com suoke-registry.cn-hangzhou.cr.aliyuncs.com

# 切换到构建目录
cd suoke-deploy/docker

# 赋予脚本执行权限
chmod +x build-images.sh

# 执行构建脚本
./build-images.sh
```

镜像版本管控规则：
- 主版本号：大改动时更新
- 次版本号：新功能时更新
- 修订号：Bug修复时更新

### 3.2 存储配置

#### 3.2.1 NFS存储配置

为了保证数据的持久性，项目使用NFS作为RAG服务的存储解决方案，配置文件位于`suoke-deploy/storage/suoke-nfs-storage.yaml`。

#### 3.2.2 RAG服务持久化

RAG服务默认将数据保存在`/data/vectors`目录，该目录通过NFS持久卷挂载，配置文件位于`suoke-deploy/ai/rag-service-with-pv.yaml`。

### 3.3 监控配置

#### 3.3.1 Prometheus + Grafana

监控系统基于Prometheus和Grafana搭建，配置文件位于`suoke-deploy/monitoring/prometheus-operator.yaml`。

主要监控指标：
- 服务健康状态
- API请求响应时间
- 资源使用率（CPU、内存）
- 容器重启次数

#### 3.3.2 ELK日志系统

日志系统基于ELK(Elasticsearch, Logstash, Kibana)实现，使用Fluentd作为日志收集器，配置文件位于`suoke-deploy/monitoring/logging.yaml`。

## 4. 部署步骤

### 4.1 前置条件

- Kubernetes集群已安装(1.21+)
- 节点已打标签(core-service, data-service, ai-service)
- 已配置阿里云镜像仓库访问密钥

### 4.2 部署顺序

1. 创建命名空间和密钥
```bash
kubectl apply -f suoke-deploy/core/aliyun-registry-secret.yaml
```

2. 部署存储资源
```bash
kubectl apply -f suoke-deploy/storage/suoke-nfs-storage.yaml
```

3. 部署核心服务
```bash
kubectl apply -f suoke-deploy/core/api-gateway-nginx.yaml
kubectl apply -f suoke-deploy/core/auth-service-simple-node.yaml
```

4. 部署AI服务
```bash
kubectl apply -f suoke-deploy/ai/rag-service-with-pv.yaml
kubectl apply -f suoke-deploy/ai/agent-coordinator-service-simple.yaml
```

5. 部署监控系统
```bash
kubectl apply -f suoke-deploy/monitoring/prometheus-operator.yaml
kubectl apply -f suoke-deploy/monitoring/service-monitors.yaml
```

6. 部署日志系统
```bash
kubectl apply -f suoke-deploy/monitoring/logging.yaml
```

### 4.3 验证部署

1. 检查Pod状态
```bash
kubectl get pods -n suoke
kubectl get pods -n monitoring
kubectl get pods -n logging
```

2. 验证API网关
```bash
kubectl port-forward service/api-gateway-nginx 8080:80 -n suoke
curl http://localhost:8080/health
```

3. 访问监控界面
```bash
kubectl port-forward service/grafana 3000:3000 -n monitoring
# 浏览器访问 http://localhost:3000
# 默认用户名: admin
# 默认密码: suoke2025
```

## 5. 维护与故障排查

### 5.1 常见问题排查

| 问题描述 | 排查方法 | 解决方案 |
|---------|---------|---------|
| Pod无法启动 | `kubectl describe pod <pod-name>` | 检查错误信息并根据情况处理 |
| 镜像拉取失败 | `kubectl get events` | 检查镜像仓库密钥和网络连接 |
| 服务无法访问 | `kubectl get svc` | 检查服务端口和选择器 |
| 持久卷问题 | `kubectl get pv,pvc` | 检查存储类和访问模式 |

### 5.2 日常维护

1. 服务扩缩容
```bash
kubectl scale deployment api-gateway-nginx --replicas=3 -n suoke
```

2. 滚动更新
```bash
kubectl set image deployment/rag-service rag-service=suoke-registry.cn-hangzhou.cr.aliyuncs.com/suoke/rag-service:1.0.1 -n suoke
```

3. 备份数据
```bash
# NFS服务器上执行
cp -r /mnt/suoke-data/rag-vectors /mnt/suoke-data/backups/rag-vectors-$(date +%Y%m%d)
```

## 6. 安全考量

- 所有服务运行在非root用户权限下
- 敏感配置使用Kubernetes Secret存储
- API通信使用TLS加密
- 服务间通信限制在必要范围内
- 定期更新依赖和基础镜像
